# Generate Pseudo Phenotype Data

## Overview

This utility generates simulated phenotype data whose sample identifiers match those of an existing expression matrix. It is useful for:

- Null-model and permutation benchmarks
- Pipeline testing without real phenotype measurements
- Generating synthetic traits for method development

Only the **header line** of each input file is read, so the full expression matrix (which can be hundreds of thousands of rows) is never loaded into memory.

## Files

```
generate_pseudo_phenotype/
├── config.yaml                   # editable parameters
└── generate_pseudo_phenotype.py  # main script
```

## Quick Start

```bash
cd code/dronetanalysis/generate_pseudo_phenotype

# Generate all datasets defined in config
python generate_pseudo_phenotype.py --config config.yaml

# Generate one dataset only
python generate_pseudo_phenotype.py --config config.yaml --dataset-tag ctrl_voom

# Override output directory and seed at runtime
python generate_pseudo_phenotype.py --config config.yaml \
    --output-dir /tmp/test_out --seed 0
```

## Config Reference

### `global` section

| Field | Type | Default | Description |
|---|---|---|---|
| `output_dir` | string | `"results/pseudo_phenotype"` | Directory where output TSV files are written. Relative paths are resolved from the config file location. |
| `seed` | int | `42` | Global RNG seed. Used for any phenotype that does not specify its own seed. |

### `datasets` list — per-dataset fields

| Field | Type | Required | Description |
|---|---|---|---|
| `input_file` | string | yes | Path to expression matrix. Relative to the config file. |
| `dataset_tag` | string | yes | Label used in the output filename: `{output_dir}/{dataset_tag}_pseudo_phenotypes.tsv` |
| `sep` | string | `"\t"` | Field delimiter of the input file. |
| `skip_first_col` | bool | `true` | When `true`, the first column of the header (typically a gene/feature ID column) is dropped before extracting sample names. |
| `phenotypes` | list | yes | One or more phenotype definitions (see below). |

### `phenotypes` list — per-phenotype fields

| Field | Type | Required | Description |
|---|---|---|---|
| `tag` | string | yes | **Unique identifier.** Written verbatim as the row name in the output TSV. Never auto-generated by the script. |
| `distribution` | string | `"gaussian"` | Sampling distribution. Currently supported: `"gaussian"`. |
| `params` | dict | yes | Distribution parameters (see below). |
| `seed` | int | no | Per-phenotype seed override. Takes precedence over `global.seed`. |

#### Gaussian parameters (`distribution: "gaussian"`)

| Param | Type | Description |
|---|---|---|
| `mean` | float | Mean of the normal distribution. |
| `std` | float | Standard deviation. |

## Function Structure

```
main(argv)
  ├─ parse_args(argv) → args
  ├─ load_config(config_path) → cfg
  ├─ resolve output_dir  (CLI --output-dir > global.output_dir)
  ├─ resolve global_seed (CLI --seed      > global.seed)
  └─ for each dataset in cfg["datasets"]:
       [skip if --dataset-tag set and doesn't match]
       process_dataset(dataset_cfg, global_seed, output_dir, config_dir)
         ├─ read_sample_names(input_file, sep, skip_first_col)
         │    └─ open file → f.readline() → split → drop col[0] if skip_first_col
         │    → list[str]          ← only ONE line read from potentially huge file
         └─ for each phenotype in dataset_cfg["phenotypes"]:
              generate_phenotype(pheno_cfg, n_samples, global_seed)
                ├─ resolve seed: pheno["seed"] if present, else global_seed
                ├─ rng = np.random.default_rng(seed)
                └─ values = DIST_GENERATORS[distribution](n_samples, params, rng)
              → (tag: str, values: np.ndarray)
         collect rows → write_output(rows, sample_names, out_path)
              pd.DataFrame(rows_dict, index=sample_names).T.to_csv(sep="\t")
```

`DIST_GENERATORS` is a module-level dispatch dict keyed by distribution name. Adding a new distribution only requires adding one entry there (see [Extending Distributions](#extending-distributions)).

## Output Format

One TSV file per dataset entry:

```
{output_dir}/{dataset_tag}_pseudo_phenotypes.tsv
```

**Shape:** rows = phenotypes, columns = samples.

```
phenotype_id       106_B2   106_B7   106_B9   ...
ctrl_body_weight   24.1     26.3     25.8     ...
ctrl_lifespan      58.2     67.1     61.4     ...
```

- The `phenotype_id` column (row names) contains exactly the `tag` strings from config.
- Column names are the sample names read from the input file header.
- The file can be used directly as phenotype input to downstream association or network analyses.

## Extending Distributions

To add a new distribution, append one entry to the `DIST_GENERATORS` dict near the top of `generate_pseudo_phenotype.py`:

```python
DIST_GENERATORS = {
    "gaussian": lambda n, params, rng: rng.normal(
        loc=params["mean"], scale=params["std"], size=n
    ),
    # Example: uniform distribution
    "uniform": lambda n, params, rng: rng.uniform(
        low=params["low"], high=params["high"], size=n
    ),
}
```

Then use `distribution: "uniform"` with the appropriate `params` keys in the config. No other code changes are needed.
